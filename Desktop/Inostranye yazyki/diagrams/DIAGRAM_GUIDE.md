# 📋 Полный список элементов для построения диаграмм LinguaAI

## 1️⃣ ДИАГРАММА ПРЕЦЕДЕНТОВ

### **Акторы (3 штуки):**
1. **Гость**
2. **Менеджер**
3. **Администратор**

### **Прецеденты (32 штуки):**

#### **Аутентификация (4):**
1. Зарегистрироваться
2. Войти в систему
3. Выйти из системы
4. Редактировать профиль

#### **Словари (5):**
5. Создать словарь
6. Добавить слово
7. Удалить слово/словарь
8. Поиск и фильтр словарей
9. Экспорт CSV/JSON

#### **Обучение (7):**
10. Просматривать карточки
11. Режим изучения (SM-2)
12. Пройти тест (4 варианта)
13. Режим сопоставления
14. Speed Round (таймер)
15. Прослушать слово (TTS)
16. Использовать горячие клавиши

#### **Прогресс (6):**
17. Просмотр прогресса
18. Получать XP за занятия
19. Поддерживать серию (streak)
20. Получать достижения
21. Просмотр таблицы лидеров
22. Получать уведомления

#### **Настройки (4):**
23. Сменить тему (темная/светлая)
24. Добавить язык обучения
25. Глобальный поиск по разделам
26. Сбросить прогресс

#### **Администрирование (6):**
27. Просмотр списка пользователей
28. Изменить роль пользователя
29. Заблокировать пользователя
30. Удалить пользователя
31. Просмотр статистики системы
32. Очистить данные обучения

---

### **Связи Актор → Прецедент:**

#### **Гость (2 связи):**
- → Зарегистрироваться
- → Войти в систему

#### **Менеджер (19 связей):**
- → Просматривать карточки
- → Режим изучения
- → Пройти тест
- → Режим сопоставления
- → Speed Round
- → Прослушать слово (TTS)
- → Создать словарь
- → Добавить слово
- → Экспорт CSV/JSON
- → Просмотр прогресса
- → Сменить тему
- → Просмотр таблицы лидеров
- → Выйти из системы
- → Редактировать профиль
- → Просмотр списка пользователей
- → Заблокировать пользователя
- → Просмотр статистики системы
- → Добавить язык обучения
- → Глобальный поиск

#### **Администратор (3 связи):**
- → Изменить роль пользователя
- → Удалить пользователя
- → Очистить данные обучения

---

### **Наследование (2 стрелки с треугольником):**
1. Менеджер --|> Гость
2. Администратор --|> Менеджер

---

### **Связи «include» (6 пунктирных стрелок):**
1. Режим изучения → Получать XP
2. Просматривать карточки → Прослушать слово (TTS)
3. Speed Round → Поддерживать серию
4. Режим сопоставления → Получать достижения
5. Создать словарь → Добавить слово
6. Пройти тест → Получать XP

---

### **Связи «extend» (3 пунктирных стрелки):**
1. Экспорт CSV/JSON → Создать словарь
2. Использовать горячие клавиши → Просматривать карточки
3. Получать уведомления → Получать достижения

## 2️⃣ ФУНКЦИОНАЛЬНАЯ СХЕМА

### **Акторы (3 штуки):**
1. **Гость**
2. **Менеджер**
3. **Администратор**

---

### **Модули/Компоненты (6 блоков):**

#### **Блок 1: Аутентификация**
1. Вход в систему
2. Регистрация
3. Управление ролями (admin/manager/guest)
4. Сессия (localStorage)
5. Выход

#### **Блок 2: Обучение**
1. Флэш-карточки (SM-2 алгоритм)
2. Режим ввода перевода
3. Тест (4 варианта)
4. Режим сопоставления
5. Speed Round (таймер)
6. TTS — озвучка слов
7. Горячие клавиши (Пробел, ←→, 1-4)

#### **Блок 3: Словари**
1. Создать словарь
2. Добавить слово
3. Поиск и фильтрация (по языку, тегам)
4. Экспорт CSV/JSON
5. Импорт CSV
6. Удалить слово/словарь

#### **Блок 4: Прогресс и геймификация**
1. XP за занятия
2. Серии (streak)
3. Достижения
4. Таблица лидеров
5. Графики активности
6. Статистика слов
7. Toast-уведомления

#### **Блок 5: Администрирование**
1. Список пользователей
2. Смена ролей
3. Блокировка/разблокировка
4. Удаление пользователя
5. Аналитика системы
6. Управление правами

#### **Блок 6: Хранилище**
- localStorage (браузер)

---

### **Связи между блоками:**

#### **От Акторов к модулям:**
- Гость → Аутентификация (регистрация, вход)
- Менеджер → Обучение, Словари, Прогресс, Администрирование (просмотр)
- Администратор → Администрирование (все функции)

#### **Между модулями:**
1. Обучение → Прогресс (начисляет XP, streak, достижения)
2. Словари → Обучение (создаёт карточки)
3. Аутентификация → localStorage (сохранение сессии)
4. Словари → localStorage (сохранение данных)
5. Обучение → localStorage (сохранение прогресса)
6. Администрирование → Аутентификация (управление ролями)
7. Прогресс → localStorage (сохранение статистики)

#### **Наследование акторов:**
- Менеджер наследует права Гостя
- Администратор наследует права Менеджера

---

## 🎨 Визуальная структура:

### **Диаграмма прецедентов:**
```
Гость ────────→ (Регистрация)
  ↑             (Вход)
  │
  │ (треугольник - наследование)
  │
Менеджер ─────→ (Карточки) ····→ (TTS)
  ↑             (Тесты)      «include»
  │             (Словари)
  │             (Прогресс)
  │             (Список пользователей)
  │
  │ (треугольник - наследование)
  │
Администратор → (Смена ролей)
                (Удаление)
                (Очистка данных)
```

### **Функциональная схема:**
```
┌──────────────────────────────────┐
│  Гость  Менеджер  Администратор  │
└────────────┬─────────────────────┘
             ↓
      ┌──────────────┐
      │ Аутентификация│
      └──────┬────────┘
             ↓
   ┌─────────┴──────────┐
   ↓                    ↓
┌─────────┐        ┌──────────┐
│Обучение │←───────│ Словари  │
└────┬────┘        └──────────┘
     │
     ├──→ Прогресс (XP, streak)
     │
     └──→ Администрирование
     
          ↓
     💾 localStorage
```

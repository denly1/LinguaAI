# 🔗 Интеграция с Bitrix24 CRM

## 📋 Описание

Бот автоматически создаёт лиды в Bitrix24 CRM при поступлении заявок от пользователей через следующие сценарии:
- 👨‍⚕️ **Вызвать доктора** (все 5 вариантов)
- 🗓 **Записаться на приём** (все 3 типа)
- 💬 **Задать вопрос администратору** (все 5 тем)

## 🚀 Настройка вебхука в Bitrix24

### Шаг 1: Создание входящего вебхука

1. Войдите в ваш Bitrix24 портал
2. Перейдите в **Приложения** → **Вебхуки** → **Входящий вебхук**
3. Нажмите **Добавить вебхук**
4. Настройте права доступа:
   - ✅ **CRM** → **crm** (полный доступ или только создание лидов)
   - Рекомендуется: только `crm.lead.add`
5. Скопируйте URL вебхука

### Шаг 2: Формат URL вебхука

URL будет выглядеть так:
```
https://your-domain.bitrix24.ru/rest/1/xxxxxxxxxxxxxx/
```

Для создания лидов добавьте метод `crm.lead.add.json`:
```
https://your-domain.bitrix24.ru/rest/1/xxxxxxxxxxxxxx/crm.lead.add.json
```

### Шаг 3: Настройка бота

1. Откройте файл `.env`
2. Найдите строку `BITRIX_WEBHOOK_URL`
3. Вставьте ваш URL вебхука:

```env
BITRIX_WEBHOOK_URL=https://your-domain.bitrix24.ru/rest/1/xxxxxxxxxxxxxx/crm.lead.add.json
```

4. Сохраните файл
5. Перезапустите бота

## 📊 Структура лида в Bitrix24

Каждый лид создаётся со следующими данными:

### Основные поля:
- **TITLE** — заголовок лида (сценарий + тема)
  - Пример: `Вызвать доктора - Запой`
- **NAME** — имя контакта (имя пользователя или username)
  - Пример: `Александр` или `@username`
- **COMMENTS** — комментарий с полной информацией о заявке
- **SOURCE_ID** — источник: `TELEGRAM`
- **SOURCE_DESCRIPTION** — описание источника: `Telegram Bot - [сценарий]`
- **OPENED** — статус: `Y` (открыт)
- **ASSIGNED_BY_ID** — ответственный (по умолчанию ID 1)

### Контактные данные:
- **IM** — Telegram username (если есть)
  - Формат: `@username`
  - Тип: `TELEGRAM`

### Пример комментария:
```
Сценарий: Вызвать доктора
Тема: Запой
Описание: Запой

Telegram ID: 825042510
Username: @username
```

## 🔧 Настройка ответственного

По умолчанию ответственным за лид назначается пользователь с ID 1.

Чтобы изменить ответственного:

1. Узнайте ID нужного пользователя в Bitrix24:
   - Откройте профиль пользователя
   - ID будет в URL: `https://your-domain.bitrix24.ru/company/personal/user/123/`
   - В данном случае ID = `123`

2. Откройте `bot.py`
3. Найдите функцию `create_bitrix_lead`
4. Измените строку:
   ```python
   "ASSIGNED_BY_ID": 1,  # ID ответственного
   ```
   на:
   ```python
   "ASSIGNED_BY_ID": 123,  # Ваш ID
   ```

## ⚙️ Дополнительные настройки

### Добавление телефона (если есть)
Если вы хотите добавить поле телефона, измените функцию `create_bitrix_lead`:

```python
payload["fields"]["PHONE"] = [
    {
        "VALUE": "+7XXXXXXXXXX",
        "VALUE_TYPE": "WORK"
    }
]
```

### Добавление воронки продаж
Чтобы лид попадал в определённую воронку:

```python
payload["fields"]["CATEGORY_ID"] = 0  # ID воронки (0 - основная)
```

### Добавление статуса
Чтобы установить начальный статус:

```python
payload["fields"]["STATUS_ID"] = "NEW"  # или другой статус
```

## 🐛 Решение проблем

### Лиды не создаются
1. **Проверьте URL вебхука:**
   - Должен начинаться с `https://`
   - Должен заканчиваться на `crm.lead.add.json`

2. **Проверьте права вебхука:**
   - Должен быть доступ к `crm.lead.add`

3. **Проверьте консоль бота:**
   - Ищите сообщения с префиксом `✗ Bitrix24:`
   - Ошибки могут быть: `timeout`, `HTTP 403`, `HTTP 404`

### Ошибка "HTTP 403"
- Недостаточно прав у вебхука
- Пересоздайте вебхук с правами `crm.lead.add`

### Ошибка "HTTP 404"
- Неверный URL вебхука
- Проверьте, что URL скопирован полностью

### Ошибка "timeout"
- Проблемы с сетью
- Проверьте доступность Bitrix24 портала

## 🔄 Отключение интеграции

Если нужно временно отключить интеграцию с Bitrix24:

1. Откройте `.env`
2. Закомментируйте или удалите строку:
   ```env
   # BITRIX_WEBHOOK_URL=https://...
   ```
3. Перезапустите бота

Бот продолжит работать, но лиды в Bitrix24 создаваться не будут.

# Telegram-бот клиники помощи зависимым

Бот для приёма обращений и передачи их администратору. Реализовано по ТЗ с поддержкой PostgreSQL.

## Возможности

- **Главное меню**: О клинике, Услуги, Вызвать доктора, Записаться на приём, Вопрос администратору, FAQ
- **Сценарии заявок**: 3 типа (Вызвать доктора / Записаться / Вопрос админу) с передачей в админ-чат
- **Режим handoff**: после передачи заявки бот игнорирует сообщения пользователя (кроме `/start`), чтобы не мешать ручной переписке
- **PostgreSQL**: все заявки, пользователи и состояния handoff сохраняются в БД
- **Редактируемые тексты**: все тексты в `texts.yml` — можно править без программиста
- **Bitrix24 CRM**: автоматическое создание лидов при поступлении заявок (опционально)
- **Админ-панель**: команда `/admin` с статистикой, активными заявками и управлением

## Установка

### 1. Установите зависимости

```bash
python -m venv .venv
.venv\Scripts\activate  # Windows
pip install -r requirements.txt
```

### 2. Настройте PostgreSQL

Создайте базу данных:

```sql
CREATE DATABASE Clinicapro;
```

Запустите SQL-скрипт для создания таблиц:

```bash
psql -U postgres -d Clinicapro -f init_db.sql
```

Или вручную:

```bash
psql -U postgres
\c Clinicapro
\i init_db.sql
```

### 3. Настройте `.env`

Скопируйте `.env.example` в `.env` и заполните:

```env
BOT_TOKEN=ваш_токен_от_BotFather
ADMIN_CHAT_IDS=825042510

DB_HOST=localhost
DB_PORT=5432
DB_NAME=Clinicapro
DB_USER=postgres
DB_PASSWORD=1

# Bitrix24 CRM Integration (опционально)
BITRIX_WEBHOOK_URL=https://your-domain.bitrix24.ru/rest/1/your_webhook_code/crm.lead.add.json
```

**Важно**: 
- `BOT_TOKEN` — получите у [@BotFather](https://t.me/BotFather)
- `ADMIN_CHAT_IDS` — ID чата(ов) администратора (через запятую, если несколько)
- `BITRIX_WEBHOOK_URL` — (опционально) вебхук для интеграции с Bitrix24 CRM. Подробнее в [BITRIX24_SETUP.md](BITRIX24_SETUP.md)

### 4. Запустите бота

```bash
python bot.py
```

Вы увидите:

```
✓ БД подключена: Clinicapro@localhost:5432
✓ Бот запущен
```

## Как работает "бот не мешает переписке"

После того, как пользователь отправил заявку (Вызвать доктора / Записаться / Вопрос админу):

1. Бот отправляет заявку администратору в Telegram
2. Пользователь переходит в режим **handoff** (сохраняется в БД)
3. Бот **игнорирует все сообщения** пользователя (кроме `/start`)
4. Администратор может вести диалог вручную (через личку пользователя или другой способ)
5. Пользователь выходит из handoff только через `/start`

## Структура БД

### Таблица `users`
- `user_id` (PK) — Telegram ID
- `username`, `first_name`, `last_name`
- `created_at`, `updated_at`

### Таблица `requests`
- `id` (PK, serial)
- `user_id` (FK → users)
- `scenario` — тип заявки (Вызвать доктора / Записаться / Вопрос админу)
- `topic` — подтема (Запой, Консультация, Стоимость и т.д.)
- `description` — текст от пользователя
- `created_at`, `status`

### Таблица `handoff_state`
- `user_id` (PK, FK → users)
- `is_handoff` — флаг (TRUE = бот остановлен)
- `handoff_at`, `cleared_at`

## Поддержка нескольких администраторов

В `.env` укажите несколько ID через запятую:

```env
ADMIN_CHAT_IDS=123456789,987654321,555555555
```

Все заявки будут приходить всем администраторам.

## Безопасность

- **Не публикуйте `.env`** в Git (добавьте в `.gitignore`)
- **Перевыпустите токен** у @BotFather, если он был скомпрометирован
- Бот **не даёт медицинских рекомендаций** и **не советует дозировки**
- В критических случаях бот напоминает: "Если есть угроза жизни — вызывайте скорую помощь (112/103)"

## Техподдержка

Если бот не запускается:

1. Проверьте, что PostgreSQL запущен
2. Проверьте, что база `Clinicapro` создана и таблицы инициализированы
3. Проверьте `.env` (токен, пароль БД, админ ID)
4. Посмотрите логи в консоли
